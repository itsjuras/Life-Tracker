# Architecture

## Overview

Life Tracker is a monorepo containing a React Native mobile app and a Node.js/Express REST API backed by PostgreSQL.

```
Life-Tracker/
├── apps/mobile/       React Native app (Expo + NativeWind)
├── services/api/      Node.js + Express REST API
├── packages/shared/   Shared TypeScript types
└── docs/              Architecture notes
```

---

## MVC Pattern

### Mobile (`apps/mobile/src/`)

| Layer      | Folder          | Responsibility                                          |
|------------|-----------------|----------------------------------------------------------|
| Model      | `models/`       | TypeScript interfaces for all domain entities            |
| View       | `views/`        | Screens and reusable UI components (React Native)        |
| Controller | `controllers/`  | Business logic; calls API services, transforms data      |

Support folders:
- `navigation/` — bottom-tab navigator config
- `services/api.ts` — thin HTTP client (fetch wrapper)
- `constants/icons.ts` — master list of selectable icon keys
- `hooks/` — small shared hooks (e.g. `useToday`)

### API (`services/api/src/`)

| Layer      | Folder          | Responsibility                                           |
|------------|-----------------|-----------------------------------------------------------|
| Model      | `models/`       | Raw PostgreSQL query functions (via `pg` Pool)            |
| View       | `views/`        | Express Router definitions — URL → controller mapping     |
| Controller | `controllers/`  | Request parsing, validation, calls model layer, responds  |

Support folders:
- `config/db.ts` — pg Pool singleton
- `middleware/` — `auth.ts` (JWT), `errorHandler.ts`
- `migrations/` — plain SQL migration files

---

## Data Flow

```
Mobile Screen
  → Controller (fetch / mutate)
    → services/api.ts (HTTP fetch)
      → Express Route (views/)
        → Express Controller (controllers/)
          → Model (models/) — SQL via pg
            → PostgreSQL
```

---

## Key Decisions

- **Dates as `DATE`** — stored without time component to keep "daily" consistent across timezones.
- **UUIDs** — all primary keys are UUIDs generated by PostgreSQL (`gen_random_uuid()`).
- **Upsert pattern** — habit entries, stat entries, and reflections all use `ON CONFLICT ... DO UPDATE` so the client can blindly call PUT/POST without checking existence.
- **npm workspaces monorepo** — `@life-tracker/shared` holds shared types importable by both the mobile app and (eventually) the web app.
